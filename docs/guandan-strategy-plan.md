# 掼蛋本地策略研发计划

## 1. 目标

本阶段不再把重点放在“让模型像会打牌一样说话”，而是把本地出牌逻辑提升到“明显具备常识和章法”的水平。

核心目标有四个：

1. 先保证规则绝对正确，任何时候不出非法牌。
2. 在合法前提下，把“最少手数”和“保关键控制牌”作为主目标。
3. 在己方协作与敌方控场之间做可解释的取舍。
4. 通过多局结构化模拟持续找反例，而不是靠肉眼看一两局。

## 2. 已收集的公开资料结论

规则口径优先采用公开规则性资料，技巧口径只提炼能转成代码的部分，不直接照搬口号。

已验证可用的信息来源包括：

1. 维基百科与其引用的 2017 版江苏竞赛规则口径：用于锚定牌型、大小、贡牌、接风、升级等基础规则。
2. Google 搜索公开文章中的常见技巧：用于提炼“记关键牌、少手数、保炸、给对家留风、对手连续控场时抢权”等软策略。
3. 公开攻略站点中关于接风与队友配合的说明：用于补充“不要挡本家”和“末手留大牌争取对家接风”的条件。

## 3. 策略优先级

### 3.1 Tier 0：硬规则

这些必须绝对满足，优先级最高：

1. 只在本地枚举的合法动作中选择。
2. 响应牌型必须可压牌。
3. 炸弹、同花顺、天王炸大小关系不能错。
4. 对家已控轮时，非收官局面不抢权。

### 3.2 Tier 1：基础常识

这些是“像会打牌”的最低标准：

1. 尽量不用大牌接小牌。
2. 尽量不拆天然炸弹。
3. 尽量少动逢人配和高控制牌。
4. 能用小的同型接牌，就不要明显过冲。

### 3.3 Tier 2：手数优化

这是当前本地策略的主优化方向：

1. 估算整手牌最少还需几手出完。
2. 领出时优先考虑能显著压缩手数的组合。
3. 在手数收益接近时，再回到“小牌先走、保控制牌”的偏好。

### 3.4 Tier 3：搭档协作

1. 对家快走完时，优先送低风险单张或对子。
2. 若有机会给对家留风，末手优先保留高把握的控牌。
3. 不为了自己短期舒服去破坏对家明显更好的节奏。

### 3.5 Tier 4：控场与抢权

1. 对手剩牌过少时，提高抢权优先级。
2. 如果对手连续两轮以上用顺子、连对、钢板之类控场，而本方一直接不住，就要提高炸弹抢权意愿。
3. 抢权不是默认动作，必须与手数收益和剩余控制资源一起判断。

## 4. 实现方式

本地策略会拆成三层：

1. 候选生成层：只负责列出合法动作。
2. 规则筛选层：先剔除明显不合理候选，例如拆天然炸弹、无必要抢对家牌权。
3. 打分决策层：综合手数、保牌、搭档、控场压力，对剩余候选排序。

## 5. 验证闭环

每次策略修改后，不只看单局演示，而是跑结构化模拟：

1. 至少 20 局完整本地对局。
2. 保存每局的发牌、行动、胜负顺序与诊断标签。
3. 自动标记明显反常动作，例如过早炸牌、明显过冲、挡本家路。
4. 根据诊断再补新规则，继续迭代。

## 6. 如果你提供书籍或人工整理内容

不要直接贴大段原文，最好整理成 Markdown 或 JSON。推荐每条规则用下面格式给我：

```md
- category: teamplay | tempo | control | opening | defense
  rule: 对家还剩 1-3 张时，优先送低单或低对
  trigger: 对家剩牌 <= 3 且当前由我领出
  action: 如果存在不拆结构的低单/低对，则优先它
  exception: 若下家已报单且我方无后续控牌，不送小单
  example: 对家 2 张，我有 6、7、A、A、炸弹，优先出 6 或对子 A 需谨慎
```

或者 JSON：

```json
{
  "category": "teamplay",
  "rule": "对家还剩 1-3 张时，优先送低单或低对",
  "trigger": "partnerCount <= 3 && isLead",
  "action": "prefer_low_safe_single_or_pair",
  "exception": "enemyReportedSingle && noControlReserve",
  "example": "partner 2 cards, hand has 6 7 AA bomb"
}
```

这样我才能稳定把资料转成可测的规则，而不是只能做泛化总结。
