# 掼蛋记牌实验室 产品与实现方案

## 1. 产品定位

这是一个面向掼蛋记牌训练的 PWA 应用。用户不直接参与出牌，而是观察一局已经预先生成完成的四人对局，通过逐手回放、被动挑战和难度升级，逐步建立以下能力：

- 入门：记住上一轮各家出过什么牌型。
- 基础：根据出牌历史，判断场上大牌还剩多少。
- 进阶：复述最近 3 轮的具体出牌内容，并回答任意点数牌还剩几张。

产品本质上不是“在线对战”，而是“掼蛋对局回放 + 记牌问答训练器”。

## 2. 用户流程

1. 打开应用后进入训练主页。
2. 系统随机生成一局完整训练牌局：两副牌、4 家、27 张/家、随机级牌、随机先手。
3. 用户通过“上一步 / 下一步”查看出牌进程。
4. 使用“下一步”推进时，系统按难度概率触发 challenge。
5. 用户回答问题后继续推进，系统累计正确率与题型表现。
6. 用户可随时结束当前牌局，或生成一局全新随机牌局重新练习。

## 3. 核心能力设计

### 3.1 对局生成

- 使用两副标准扑克牌加大小王，共 108 张。
- 四家固定座位：南、东、北、西，对家为队友。
- 每局随机决定当前级牌。
- 红桃级牌按“逢人配”处理，可在组合牌中充当任意非王牌。
- 开局即预生成整局回放轨迹，用户只负责观察与训练。

### 3.2 出牌规则

当前版本重点保证“单局内出牌过程”符合真实掼蛋思路，覆盖：

- 单张、对子、三同张、三带二、顺子、三连对、钢板。
- 4/5 张同点炸、6 张及以上同点炸、同花顺、天王炸。
- 炸弹可压普通牌型，炸弹之间按真实层级比较。
- 三家不再压住时，上一手有效出牌者收轮并重新领出。
- 若收轮者已走完，则由其对家优先领出。

### 3.3 规则边界

为了让训练聚焦在“记牌”而不是跨局积分结算，V1 不做以下跨局机制：

- 升级进贡
- 报牌交互
- 多局累计胜负

这些内容不影响单局内记牌训练，也不影响出牌合法性。

### 3.4 AI 出牌策略

AI 不是穷举最优博弈，而是“合法且像真人”的启发式策略：

- 优先拆出顺子、钢板、三连对、三带二等高效率结构。
- 保留大炸与天王炸，不轻易裸炸。
- 队友在当前轮领先时，默认让牌，不随意反超。
- 对手剩牌危险时，提高压制与炸弹使用倾向。
- 临近出完时，优先选择能缩短残局的高控制力牌型。

### 3.5 记牌训练题库

题库按难度分层：

- 入门：上一轮谁出了什么牌型、谁收轮。
- 基础：A、级牌、大小王、逢人配还剩多少张。
- 进阶：最近 3 轮的具体内容、任意点数牌的实体剩余张数。

问答采用选择题，保证移动端上手成本低。

## 4. 信息架构

- 牌桌区：四家剩牌数、当前轮、当前压住方。
- 回放区：当前轮动作流与最近三轮摘要。
- 训练区：难度切换、挑战弹窗、答题统计。
- 控制区：上一步、下一步、结束本局、新牌局。

## 5. 技术实现

### 5.1 前端栈

- React 19
- TypeScript
- Vite
- 原生 manifest + service worker，构成可安装 PWA

### 5.2 规则引擎

- 使用结构化牌模型表示实体牌、级牌与逢人配。
- 在手牌分析阶段拆分：自然牌、逢人配、顺子用牌池、同花顺用牌池。
- 生成当前手牌所有合法候选出法，再由启发式策略筛选。
- 在牌局生成阶段一次性模拟整局，得到完整动作序列。

### 5.3 持久化

- 使用 localStorage 保存当前牌局、步进位置、答题统计与难度。
- 页面刷新后可继续当前训练。

## 6. 美术与交互方向

- 风格：纸牌俱乐部 + 训练台账，不走博彩感。
- 配色：墨绿牌桌、暖白卡片、铜金强调色。
- 视觉重点：四家座位关系清晰、当前轮聚焦强、题目弹窗不打断方向感。
- 移动端优先：单手点击区大、历史信息分块、统计可折叠。

## 7. 部署方案

- 使用 GitHub Actions 在 push 到 main 时自动构建并部署到 GitHub Pages。
- Vite 的 base 由 Actions 环境自动切换，兼容 Pages 子路径部署。
- 仓库内补充工作流文件后，只需绑定远端仓库并开启 Pages 即可生效。
